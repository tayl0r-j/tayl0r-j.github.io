Original Artifact

Python/Dash Dog Shelter Dashboard

The original artifact was a data visualization dashboard built with Python, Dash, and Plotly. It was designed to help animal shelter staff track and manage dog intake, adoption rates, and shelter capacity. The application featured interactive charts, filtering capabilities, and basic CRUD operations connected to a MongoDB database.

While functional, the artifact had several limitations:

  • Limited scalability due to monolithic architecture
  • No clear separation between business logic and presentation
  • Minimal validation and error handling
  • No comprehensive test coverage
  • Single-user design with no authentication or authorization

This artifact served as the foundation for demonstrating growth across multiple computer science categories.

Enhancements & Domain Pivot

Transformation: From Dog Dashboard to Raid Tool Command Center

Rather than making incremental refinements to the original Python/Dash application, I made a deliberate decision to re-design the system and pivot it to a domain I understand deeply: raid management for WoW: Classic. This shift allowed me to approach the project from a true stakeholder perspective while preserving and strengthening the core computer science objectives of the capstone.

The enhanced system was rebuilt as a full-stack Java Spring Boot application backed by a normalized PostgreSQL database. I introduced a relational data model to represent raid teams and members, implemented JPA repositories and service-layer logic to enforce data integrity, and exposed RESTful endpoints to support reliable CRUD operations and sorted roster retrieval. These enhancements significantly improved data consistency, query performance, and overall maintainability compared to the original implementation.

This domain pivot enabled me to demonstrate the same foundational skills required by the original artifact in a context where I could reason like an end user and system owner. The result is a more meaningful, production-oriented artifact that reflects not only technical proficiency, but also thoughtful system design, stakeholder awareness, and professional software engineering practice.

Enhancement 1: Software Design & Engineering

Migration to Java Spring Boot Architecture

I migrated the entire application from Python/Dash to Java with Spring Boot, implementing:

  • RESTful API architecture: Clean separation of concerns with controllers, services, and repositories
  • Dependency injection: Spring's IoC container for testable, maintainable code
  • Layered architecture: Clear boundaries between presentation, business logic, and data access
  • Error handling: Global exception handling with meaningful HTTP status codes and responses
  • Input validation: Bean validation for request DTOs

Enhancement 3: Databases

PostgreSQL Database Design & Optimization

I redesigned the data layer from MongoDB to PostgreSQL, implementing:

  • Normalized relational schema: Players, Characters, Raids, RaidRoster, LootDistribution tables
  • Foreign key constraints: Ensuring referential integrity
  • Indexing strategy: Optimized queries for common lookup patterns
  • Transactions: ACID-compliant operations for roster changes and loot assignments
  • JPA/Hibernate integration: ORM with Spring Data repositories for clean data access
